# Demo Script

## Goal

Show the transformation from noisy PR context to a short, high-confidence checklist.

## 30-60 second flow

1. Open on a PR-shaped input
- show a PR URL or `owner/repo#number`
- briefly hint at diff, comments, and checks

2. Transition into review noise
- visually stack changed files, review comments, and failing or passing checks
- communicate complexity without requiring exact realism

3. Snap into structure
- reveal the checklist sections in this order:
  - Key Changes
  - Risks
  - Questions
  - Suggested Tests

4. Close on confidence
- show a concise final state that feels actionable
- optional secondary frame: `Reply Draft`

## Visual guidance

- Favor contrast between messy input and clean output
- Treat the checklist as the hero artifact
- Make the animation or layout feel intentional, not dashboard-generic
- Avoid copying GitHub's full interface literally; use recognizable cues instead
