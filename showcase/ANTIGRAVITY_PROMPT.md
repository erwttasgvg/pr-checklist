# Prompt For Antigravity

Use this prompt as-is or adapt it slightly.

```txt
Design the presentation layer for a GitHub project called "GitHub PR Checklist".

Context:
- This is a single-purpose Codex skill, not a full platform.
- The product turns a GitHub pull request into an actionable review checklist.
- Core promises:
  1. Review faster
  2. Miss fewer risky changes
  3. Avoid fluffy summaries
- Inputs:
  - GitHub PR URL
  - owner/repo#number
  - fallback diff/patch
- Output structure:
  - Key Changes
  - Risks
  - Questions
  - Suggested Tests
  - optional Reply Draft

What I need from you:
- Create a high-taste GitHub project presentation direction.
- Focus on README showcase structure, hero treatment, section hierarchy, visual tone, and demo/screenshot ideas.
- Make it feel sharp, technical, and memorable.
- Do not redesign the product logic or invent new features.
- Do not turn it into a generic software landing page.
- Avoid purple-on-white default automation aesthetics.
- Avoid bloated dashboards.

Desired deliverables:
- A recommended GitHub README presentation format
- A visual concept for the hero section
- A section-by-section showcase structure
- Suggestions for mock screenshots or demo frames
- Color, typography, and composition direction
- Optional social preview / Open Graph image concept

Use the content and constraints from:
- showcase/content/landing-copy.md
- showcase/content/showcase-brief.json
- showcase/content/demo-script.md

Design target:
- It should look like a project people want to star because it feels unusually clear, useful, and well-packaged.
- The presentation should emphasize the transformation from PR noise to decision-ready checklist.
```
